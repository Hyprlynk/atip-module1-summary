Basic Text Classification

The second tutorial teaches binary text classification using an IMDB dataset. The task is to determine whether the sentiment of a movie review is positive or negative. So it seems like a simpler task than the first tutorial, but the dataset requires more preprocessing, making it a bit more complicated.

Preprocessing includes standardization, tokenization, and vectorization:

  • standardize
    • preprocesses the text to remove things like html tags and punctuation
  • tokenize
    • splits the strings into tokens (e.g. splitting dataset into words by whitespace)
  • vectorize
    • converts tokens into numbers so they can feed into the neural network

We create a custom standardization function:

def custom_standardization(input_data):
  lowercase = tf.strings.lower(input_data)
  stripped_html = tf.strings.regex_replace(lowercase, '<br />', ' ')
  return tf.strings.regex_replace(stripped_html,
                                  '[%s]' % re.escape(string.punctuation),
                                  '')

Which is then used in a vectorization layer, which will do all of the preprocesing:

vectorize_layer = TextVectorization(
    standardize=custom_standardization,
    max_tokens=max_features,
    output_mode='int', # creates unique integer indices for each token
    output_sequence_length=sequence_length)

Deep Dream

For the third tutorial, I decided to look into some more advanced techniques. Deep Dream is a generative technique where a pre-trained image classification network looks at an image and "hallucinates" patterns in the image based on the data it was trained on.

This technique requires a lot of complicated code, so I'll just try and summarize the parts I understand.

  1. First, pick an image to give to the network.
  2. Then, choose a network to use. I started with the InceptionV3 network.
  3. Choose a couple of specific layers from that network.
  4. Calculate the loss of those layers.
  5. Normally, you would want to minimize loss using gradient descent, but here you try to maximize loss using gradient ascent.
  6. Repeat, adding the loss back onto the original image, which changes the image a little bit, causing the layers to become more excited and patterns to emerge. (This is the part that is kind of weird and magical.)

You can also start with a small image, run the algorithm, and scale it up a few times (called octaves), which produces more detailed and vivid patterns. I used the tutorial's code to do this, because it makes your output much better.

Choosing my own dataset

For a "final project" of sorts, I decided to download a new dataset from the internet and see how far I could get without following any particular tutorial.

I found the UCI Machine Learning Repository, which has a large number of datasets for download, and I downloaded this dataset:

https://archive.ics.uci.edu/ml/datasets/Connect-4

The dataset "contains all legal 8-ply positions in the game of connect-4 in which neither player has won yet, and in which the next move is not forced... The outcome class is the game theoretical value for the first player."

The task is to classify whether a given position would result in a win, loss, or draw for player 1.

So I had the dataset and I spent some time figuring out how to load it into a data structure. Eventually I found some instructions on how to load CSVs, and I loaded the data into a pandas dataframe:

connect4_data = pd.read_csv(
    data_csv,
    header=None)

Then I had to figure out how to split the data into train and test data, and since I had never done that using a pandas dataframe before, I looked around and found this function:

from sklearn.model_selection import train_test_split
# 80:20 split of training and test data

Next, I struggled for a while with how to preprocess the data. The dataset consists mostly of the letters 'b,' 'x,' and 'o,' representing a blank connect 4 cell, or cell occupied by a player's piece. The outcome classes are strings: "win," "loss," or "draw." I tried to tokenize or vectorize the text somehow, but none of the methods I tried would accept the data in the format I had it in. Eventually, I decided to just use ctrl+f and replace, changing the letters to integers 0, 1, or 2, and the same for the output classes.

Now that I had the data in the right format, I could easily load it into a simple model.
